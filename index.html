Intro

Do you want to embed a 360 video in your webpage? Something like:

.... but controllable?

There are tools out there, that help you create these videos (should you have the hardware for it)

  • autopano video from kolor 200$+
  • video stitch 200$+ (free with watermark)
  • PTGui 100$+
  • more info here

But noone really offers (free/open) solutions on how to showcase these on your website

DEMO

Here you can view a DEMO that runs on Chrome, Firefox on Windows, Linux and OSX.

Controls

  • drag the image around to pan
  • space to pause the video
  • p to play
  • r to rewind
  • m for fullscreen

You could embed this into your page as well, you'd just have to check out the example directory of the github project

The master branch of the project contains a meteor.js project with the same capabilities as the demo above

You can download the zip or tar version and run & play around with the sample. Note though: for good performance use chrome, and if you run it locally (from file:///) then start chrome wiht the --allow-file-access-from-files flag, otherwise the video won't be loaded.

The plan

My goal would be - given enough free time - to develop a small framework that would be user/dev friendly that would allow

  • Easy embedding of a 360 video into a web page
  • Nice UI controls for the "player"
  • A small offline app to convert the video into a suitable web format (for multiple platforms)
  • API for controlling
    • Looking direction
    • Quality
    • Field of view

How this was done

  • I've downloaded the sample output of the video-stitch software found here, and while looking at it realized that if mapped to a sphere it would give a nice view
  • Afterward I've found this three.js based project showing how you can map videos to webgl objects
  • I've converted the mp4 - which for some reasons didn't play in the browser - to webm with ffmpeg 
    ffmpeg -i ./in.mp4 -c:v libvpx -b:v 10M -c:a libvorbis ./out.webm
    - 10M is for the bitrate of the output video
  • Then I've just fiddled around with the code, adjusting camera position and using a sphere as the output et voil√†

Motivation

I am working on an awesome project, in which we'd like to embed 360 videos in a page, and we've got a ridiculously expensive offer from one of the companies that provide such facilities for a limited functionality that we wanted.

After a bit of thinking / googling around I've realised that is not that hard and in an evening just sat down, and threw the project together in 2 hrs.

I believe that for limited scenarios (eg: where no cross platform/browser/mobile support is needed) this solution should be publicly available & free of charge. Therefore I am publishing this, maybe it will help someone out, and as mentioned maybe me or someone else can develop it further into a generic solution
